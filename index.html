Technical Work

CampusFix

CampusFix is a campus issue reporting and tracking mobile application built for Android using Java and Firebase. The app allows students to report campus problems such as broken equipment, safety concerns, and maintenance issues, while staff users can triage, assign, track, and resolve them through a role-based workflow. The project was developed across four agile sprints, starting with Firebase authentication, MVVM architecture, and role-based navigation, then progressing through Firestore-backed issue creation and feed display with filtering and visual indicators. Later sprints added a full issue lifecycle with defined status transitions (Submitted through Closed), a staff dashboard for assignment and updates, an analytics dashboard with live Firestore-powered charts, and an external API integration. The app also features issue upvoting, robust error handling for offline and permission-denied scenarios, and multiple clearly documented design patterns governing issue state and behavior.

Java Android Studio Firebase Auth Firestore MVVM REST API

Local Explorer

A JavaFX desktop application that displays real-time weather conditions, local news headlines, and upcoming event listings based on a user-provided location. The app connects to three external APIs (OpenWeatherMap, NewsData, and Ticketmaster) using multithreaded calls, fetching and rendering data concurrently so the UI remains responsive. This project deepened my understanding of working with REST APIs, managing asynchronous tasks in Java, and building polished desktop interfaces with JavaFX.

iTunes Media Gallery

A JavaFX application that queries the iTunes Search API and displays a gallery of 20 media images based on user search terms and a selected media type. The app leverages multithreading to download images concurrently, updating a progress bar in real time without freezing the UI. Building this project reinforced key concepts around concurrency, API consumption, and responsive GUI design in Java.
